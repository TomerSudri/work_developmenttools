# פרויקט Drupal עם Docker

## חברי הפרוייקט: תומר אסחייק, אגם סובל ולורן ספר
## מה נדרשנו לעשות
במסגרת פרויקט זה נדרשנו להקים אתר Drupal באמצעות Docker, לשמור את הנתונים בבסיס נתונים MySQL, ליצור סקריפטים לניהול המערכת לגיבוי ושחזור, ולהעלות הכל ל-Git repository בשם "tomer_agam_loren_devtools".

## מה עשינו
1. הקמנו סביבת Docker עם רשת משותפת לקונטיינרים
2. התקנו והגדרנו קונטיינר MySQL עם סיסמת root: my-secret-pw
3. התקנו והגדרנו קונטיינר Drupal עם חשיפת פורט 8080
4. יצרנו אתר בשפה עברית/אנגלית עם שם "האתר של [שמות חברי הצוות]"
5. יצרנו חשבון מנהל demoadmin עם סיסמה secretpass וחשבונות לכל חברי הצוות
6. הוספנו תכנים לאתר מתוך מילון המושגים של הקורס
7. פיתחנו סקריפטים לאוטומציה של ההקמה, גיבוי, שחזור וניקוי

## הטכנולוגיות בהן השתמשנו
- Docker - למיכול השירותים
- Drupal - מערכת ניהול תוכן
- MySQL - בסיס נתונים
- Bash - שפת סקריפטים
- Git - ניהול גרסאות

## מדריך צעד אחר צעד לשחזור האתר

### דרישות מקדימות
- מערכת הפעלה Linux
- Docker מותקן
- Git מותקן

### שלב 1: הורדת הקוד מהביט באקט
git clone https://ashaiektomer@bitbucket.org/tomer_agam_devtools/tomer_agam_loren_devtools.git
cd tomer_agam_loren_devtools

### שלב 2: הרשאות הרצה לסקריפטים
chmod +x setup.sh backup.sh restore.sh cleanup.sh

### שלב 3: הקמת סביבת העבודה
./setup.sh

הסקריפט יבצע את הפעולות הבאות:
- יצירת רשת Docker
- הפעלת קונטיינר MySQL
- הפעלת קונטיינר Drupal
- הצגת הוראות להמשך הקינפוג

### שלב 4: קינפוג Drupal
1. פתח דפדפן וגש ל-http://localhost:8080
2. עקוב אחר ההוראות להגדרת Drupal:
   - בחר שפה (עברית או אנגלית)
   - בחר פרופיל התקנה (Standard)
   - הגדר את חיבור בסיס הנתונים:
     - סוג בסיס נתונים: MySQL
     - שם השרת: drupal_mysql
     - שם בסיס הנתונים: drupal
     - שם משתמש: drupal
     - סיסמה: drupal
   - כשנשאל על שם האתר, יש לכתוב בדיוק: "האתר של [שמות חברי הצוות]"
   - יצירת חשבון מנהל:
     - שם משתמש: demoadmin
     - סיסמה: secretpass
   - יצירת חשבונות נוספים לכל אחד מחברי הצוות

### שלב 5: יצירת תכנים באתר
1. התחבר לאתר עם המשתמש demoadmin והסיסמה secretpass
2. גש למסך הניהול של Drupal
3. הוסף תכנים חדשים מתוך מילון המושגים המופיע באתר המודל של הקורס
4. עצב את האתר לפי רצונך (אופציונלי)

### שלב 6: גיבוי המערכת
./backup.sh
הסקריפט יבצע:
- גיבוי של בסיס הנתונים MySQL לקובץ my-drupal.backup.sql.gz
- גיבוי של קבצי האתר לקובץ drupal_files_backup.tar.gz
- שמירת הקבצים בתיקיית backup

### שלב 7: שחזור המערכת (במכונה אחרת)
1. התקן את הסביבה במכונה החדשה באמצעות setup.sh
2. העתק את קבצי הגיבוי לתיקיית backup
3. הרץ את סקריפט השחזור:
./restore.sh
הסקריפט ישחזר את בסיס הנתונים וקבצי האתר מהגיבוי.

### שלב 8: ניקוי סביבת העבודה
./cleanup.sh
הסקריפט ינקה את כל המשאבים:
- עצירת ומחיקת קונטיינרים
- מחיקת Volumes
- מחיקת רשת Docker
- מחיקת תמונות Docker
